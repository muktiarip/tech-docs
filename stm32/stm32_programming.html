# STM32 Programming Notes

# Starting STM32 Programming

Its always best to start with an existing project, then adapt that. If you don't have a suitable existing project then I suggest:

  1. Create a makefile that builds a simple hello world style application for your hardware - without adding in the FreeRTOS source files yet.
  2. Ensure your project builds and runs as expected. That will give you a known good starting point before you add in more source code.
  3. Add the FreeRTOS source files to your project. You will find information on the files that are needed, and the include paths that must be set, on the following page: http://www.freertos.org/Creating-a-new-FreeRTOS-project.html . The port layer files you need are in the FreeRTOS/source/portable/gcc/ARM_CM0 directory (assuming this is a Cortex-M0 part?).
  4. You will need a FreeRTOSConfig.h file. The one used by the official FreeRTOS STM32F051 demo will probably be a good starting point.
  5. Make sure the project builds, and you can still run the hello world application.
  6. Now create your FreeRTOS tasks - here is a good place to start: Hardware independent FreeRTOS example

# Flashing BluePill using openocd

openocd \
--file /data/embed/openocd-conf/stm32-bluepill.cfg \
--command "program build/spi_a.elf verify reset exit"

# Enable/Disable SWD Debugging

In file stm32f1xx_hal_msp.c:

__HAL_AFIO_REMAP_SWJ_NOJTAG(); -- Enable SWD debugging.

__HAL_AFIO_REMAP_SWJ_DISABLE(); --> Disable SWD debugging

# Debugging STM32 App

  • Compile with -g option of gcc compiler.

  • In terminal #1:

openocd -f /data/embed/openocd-conf/openocdstm32-bluepill.cfg
  • In terminal #2:
/venv/gdbgui/bin/gdbgui -n -g \
  /embed/gcc-arm-none-eabi-8-2018-q4-major/bin/arm-none-eabi-gdb \
  path/to/stm32_program.elf
  • Using web browser:
    • Open URL: http://127.0.0.1:5000
    • In gdb command line, type:
      • target remote localhost:3333
      • monitor reset halt
      • continue
    • Add breakpoints as necessary using breakpoint gdb command

# STM32F1 Unbrick

  • Hold reset button while booting.
  • Connect openocd to target
  • On another terminal, telnet to localhost:4444, then send command reset halt
  • Release reset button.
  • On telnet session, send command stm32f1x mass_erase 0, then exit

If you unable to program STM32F1 chip, hold reset button before flashing, run flashing software and immediately release reset button.

# Debugging STM32 Application using Qemu

  • Compile with -g option of gcc compiler.
/data/venv/python3.6/bin/gdbgui -n -g \
    /data/embed/gcc-arm-none-eabi-8-2018-q4-major/bin/arm-none-eabi-gdb \
    demos/uart_echo_int/main.elf

/usr/local/bin/qemu-system-arm -M stm32-p103 -gdb tcp::3333 -S \
    -kernel demos/uart_echo_int/main.bin
  • Open web browser to url: http://localhost:5000/ then gdbgui/> connect to localhost:3333

# SD Card Module

  • Blue MISO -> BPL#31:PA6
  • Green SCK -> BPL#30:PA5
  • Red MOSI -> BPL#32:PA7
  • Brown CS -> BPL#29:PA4
  • Yellow 3.3 volt -> BPL#
  • Orange GND -> BPL#
